First assignment: build and analyse your graph

In the first assignment we work with graphs that can be:

  1. produced by using public APIs (see for example the list of APIs at http://www.programmableweb.com/apis/directory/1?sort=mashups);
  2. syntetic, i.e. produced applying one of the network generation models discussed in the class;
  3. already published in online repository, see for example the web graphs in the Stanford Large Network Dataset Collection.

Once you have your graph, you should create a text file describing it in a format that the software Gephi supports (see https://gephi.org/users/supported-graph-formats/).

Open the graph with Gephi, see its appearance and compute the measures we have discussed during lectures. Then write a short report describing your graph and the results you have obtained.

You can select any domain (e.g. twitter, lastFm, Wikipedia, etc.) and you can use the programming language/framework of your choice. I know there is already a lot of software available in the "huge web" and you can reuse whatever you like, but you should understand what is going on.

Note: in the Stanford pages I read:

"Stanford Network Analysis Platform (SNAP) is a general purpose network analysis and graph mining library. It is written in C++ and easily scales to massive networks with hundreds of millions of nodes, and billions of edges. It efficiently manipulates large graphs, calculates structural properties, generates regular and random graphs, and supports attributes on nodes and edges."

and also:

"Snap.py is a Python interface for SNAP. It provides performance benefits of SNAP, combined with flexibility of Python. Most of the SNAP C++ functionality is available via Snap.py in Python."

I do not know this library, but if anyone is curious and wants to use it, is free to do so. This assignment is "democratic" like random graphs :)
